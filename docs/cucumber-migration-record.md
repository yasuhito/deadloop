# Cucumber 移行記録

この記録は [分類表](cucumber-test-classification.md) の ID と受け入れ仕様の対応を追跡する。受け入れシナリオで完全に置換した Vitest だけを削除し、外部観測可能な境界を用意できない保証は Vitest 継続へ再分類する。

## Issue #128: 公開設定と安全な既定値

Issue #128 で対象にした T076〜T133 は Vitest 継続へ再分類した。今回作成した設定シナリオは、設定の探索・正規化・解析結果を直接検査しており、エージェント起動、表示、自動化判断などの利用者が観測できる結果まで到達していなかった。このため受け入れ仕様として採用せず、削除した `test/core.test.ts` のテストを復元した。

設定が実際の起動や自動化判断に反映された結果を決定論的なテスト用アダプター越しに観測できる境界を用意するまでは、次の扱いとする。

- T076〜T081、T093〜T101、T103、T104、T108〜T111、T118〜T120、T122、T123、T125 は、設定の探索・正規化・共有方針との結合を局所的に診断できる既存 Vitest を継続する。
- T082〜T092、T102、T105〜T107、T112〜T117、T121、T124、T126〜T133 は、当初の記録どおり、解析、時刻計算、文字列整形、テンプレート、呼出引数などの低レベル契約を診断する Vitest を継続する。
- T079 と T110 の一部だけを Cucumber へ移したという扱いも取り消す。対応する既存 Vitest が契約全体を引き続き検査する。

したがって、Issue #128 で Cucumber 移行済みまたは同じ保証へ統合した分類 ID はない。
