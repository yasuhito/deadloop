# 機能: deadloop doctor が停止、安全性、占有の異常を表示する

オペレーターが自動化を安全に復旧できるよう、問題だけを検出し、確認または復旧のための情報を示す。

## シナリオ: 停止した Issue に再投入コマンドを表示する

* 前提 Issue が停止状態である
* もし deadloop doctor を実行する
* ならば 停止した Issue の再投入コマンドが表示される

## シナリオ: 停止した Issue の最新の理由を要約する

* 前提 Issue が停止理由のコメントを持つ
* もし deadloop doctor を実行する
* ならば 停止理由が表示される

## シナリオ: 長時間更新されない作業中 Issue の作業場所を確認できる

* 前提 作業場所を持つ古い作業中 Issue がある
* もし deadloop doctor を実行する
* ならば 作業場所の変更を確認するコマンドが表示される

## シナリオ: 最近更新された作業中 Issue を問題として表示しない

* 前提 担当者が稼働中の最近更新された作業中 Issue がある
* もし deadloop doctor を実行する
* ならば 問題は表示されない
